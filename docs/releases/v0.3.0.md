# DreamSkin Studio v0.3.0

这是 DreamSkin Studio 第一套可安装的 macOS Apple silicon 桌面发行版。Studio、主题中心、
Trae/WorkBuddy 适配运行时和本地 Agent 连接均包含在应用中，不需要 Node.js，也不需要单独
启动 MCP Server。

## 本版亮点

- 以 DMG 交付，拖入“应用程序”即可安装；ZIP 作为备用下载格式。
- 主题中心内置 9 套模板，并支持把模板添加到“我的主题”。
- 可以分别为 Trae 和 WorkBuddy 新建空白主题。
- 连接本机 Codex CLI 后，可以通过对话生成和修改当前主题。
- 编辑器同步展示首页、对话页、结果页及常用组件状态，而不只是颜色配置。
- 应用、验证和恢复原生界面均为明确操作；退出 Studio 不会擅自撤销已应用主题。
- 运行状态会核对目标进程、注入器、CDP 与 owned launchd job；残留记录不会再被误报为已连接。
- 首次启动会在应用数据目录安装经过清单校验的 Trae 与 WorkBuddy 运行时。
- 设置页提供软件更新状态；签名发行版可以检查、下载并在确认后重启安装新版本。

`v0.3.0` 是 stable channel 发行版。自动更新元数据固定为 `latest-mac.yml`，其中同时记录 ZIP 与
DMG，Release 还会附带两种载荷各自的 blockmap。本轮发布流程不接受 prerelease 版本；后续若
开放 beta/alpha，会使用独立 channel 和对应更新清单，不与稳定版混用。

## 安装

1. 下载 `DreamSkin-Studio-0.3.0-mac-arm64.dmg`。
2. 打开 DMG，把 **DreamSkin Studio** 拖到 **Applications**。
3. 从“应用程序”启动 DreamSkin Studio。
4. 如需对话修改主题，在 Studio 中连接本机可用的 Codex CLI。

正式 Release 同时提供 `SHA256SUMS.txt`。下载后可以执行：

```bash
shasum -a 256 -c SHA256SUMS.txt
```

正式 DMG 必须显示 Developer ID 签名并通过 Apple 公证。发行链会对最终 DMG 单独执行
`notarytool submit --wait`，只有返回 `Accepted` 后才 staple 票据并交给 Gatekeeper 验证；仅公证
DMG 里的应用本体不算完成。Actions 中的 unsigned artifact 仅用于受控测试，不是公开发行版。

## 兼容性

- macOS 12 或更高版本。
- Apple silicon（arm64）。
- Trae 当前实机适配基线：TRAE SOLO CN `0.1.36`。
- WorkBuddy 当前实机适配基线：`5.2.6`。
- 对话修改主题需要本机 Codex CLI；浏览、编辑、应用和恢复主题本身不依赖云端 Studio。

## 数据与升级

主题、版本化运行时、备份和 Studio 状态保存在：

```text
~/Library/Application Support/DreamSkin Studio/dreamskin/
```

从早期本地预览版首次升级时，Studio 会把旧 Electron 预览版与 Web Studio 的主题、主题库、
预览图和事务备份合并到新目录。迁移不覆盖新目录中已经存在的内容；每一项成功后都会记录，
临时失败的项目会在下次启动单独重试，已经迁移后又被用户删除的主题不会重新出现。旧目录不会
自动删除，避免迁移失败造成数据丢失。

Trae 与 WorkBuddy 的已应用主题状态分别保存在稳定的本机运行目录中，更新 Studio 不会改变这些
目录。使用 `--user-data-dir` 启动的隔离验收实例只允许浏览和编辑隔离数据，不允许应用、预览、
验证或恢复真实目标软件，避免测试实例接管全局运行会话。

更新应用不会主动恢复已经应用到 Trae 或 WorkBuddy 的主题。需要退出主题时，请在 Studio
设置中选择对应目标并执行“恢复原生界面”。

## 已知限制

- 本版没有经过 Intel Mac 验收，不提供 x64 或 universal 安装包。
- Windows 桌面安装包仍在验证阶段，不属于本次发行范围。
- Trae 或 WorkBuddy 升级后，如果其界面结构或签名身份发生变化，DreamSkin 可能先停止应用
  主题并要求更新适配，而不是继续进行不安全注入。
- 自动更新只对 Developer ID 签名并完成 Apple 公证的正式 Release 开放；ad-hoc 测试包不会
  自动下载安装更新。
- 当前打包 Agent 连接器为 Codex CLI，其他 CLI Agent 将在后续版本中扩展。

## 验收范围

发行门禁覆盖源代码检查、全量自动化测试、Studio production build、资源清单、Electron
fuses、打包态 UI、Trae/WorkBuddy 运行时、DMG/ZIP 载荷一致性、拖拽安装后启动、Developer
ID、Gatekeeper 和公证票据。正式 Release 只有在所有适用门禁通过后才会从草稿转为公开。
